## Phase 4: Notifications via Service Bus

**Acceptance Criteria:** When an order is confirmed, an email-worker function receives the event and logs the confirmation (simulated email). When an order expires, an expiry-worker function receives and logs that too.

**Azure Resources to Add:**

- Service Bus Namespace
- Service Bus Topic: `order-events`
- Subscriptions: `email-worker`, `analytics-worker`

**What you build:**

- Publish to Service Bus topic from within the orchestrator (confirmed + expired)
- `email-worker` Function — Service Bus trigger, logs confirmation
- `analytics-worker` Function — Service Bus trigger, logs event for analytics
