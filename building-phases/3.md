## Phase 3: Reservation Expiry

**Acceptance Criteria:** If an order is left in a pending state for X minutes (configurable per-event), the seat is automatically released and the order is marked as expired — with no manual intervention.

**Azure Resources to Add:**

- None _(Durable Timer runs within existing Functions infrastructure)_

**What you build:**

- Durable timer running in parallel from the moment a seat is reserved
- Compensation activity: increment `availableSeats` back, update order → `expired`
