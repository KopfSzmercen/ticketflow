## Phase 1: Event Management + IaC & CI/CD Baseline

**Acceptance Criteria:** As an organizer I can create an event and retrieve a list of all events via API (Postman/curl, no UI yet). All resources are provisioned exclusively via Bicep — no manual portal clicks. A GitHub Actions pipeline deploys the Function App on every push to main.

**Azure Resources to Add:**

- Resource Group
- Azure Functions App (Consumption plan)
- Storage Account _(required by Functions runtime)_
- Cosmos DB Account + Database + `events` container

**What you build:**

- Bicep templates for all resources from day one
- GitHub Actions workflow: build → test → deploy Functions
- Environment variables / app settings wired via Bicep
- `POST /events` — create event
- `GET /events` — list all events
- `GET /events/{eventId}` — get single event

> Start with Bicep before writing a single line of application code. It takes an hour and saves many hours of retrofitting later.
